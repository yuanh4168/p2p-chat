# P2P 加密聊天 —— 去中心化 · 端到端加密 · 无需服务器

一款基于 **UDP + DHT** 的纯 P2P 聊天软件，没有中央服务器，所有消息点对点传输，并经过端到端加密（Ed25519 + X25519 + AES-256-GCM）。支持创建群组、话题、成员管理，并可通过 **Tailscale** 或公网 IP 直接连接。

## 项目优势（与其他聊天软件对比）

| 特性 | 本软件 | 传统中心化聊天（微信/QQ） | Matrix/Signal（去中心化） |
|------|--------|---------------------------|---------------------------|
| **服务器依赖** | ❌ 无 | ✅ 必须依赖中心服务器 | ❌ 依赖家庭服务器或第三方 |
| **端到端加密** | ✅ 默认所有消息加密 | ❌ 部分加密，服务端可读 | ✅ 加密，但依赖服务器中转 |
| **身份控制** | ✅ 基于 Ed25519 密钥对，完全自主 | ❌ 手机号/第三方账号 | ✅ 密钥对，但需服务器验证 |
| **网络穿透** | ✅ 内置 DHT 自动发现节点 | ❌ 需公网服务器转发 | ❌ 依赖 TURN/STUN |
| **Tailscale 原生支持** | ✅ 直接使用 Tailscale IP 连接 | ❌ 不支持 | ❌ 需要额外配置 |
| **群组管理** | ✅ 去中心化群组，管理员批准加入 | ✅ 群组由服务器管理 | ✅ 需服务器协调 |
| **离线消息** | ❌ 不支持（纯 P2P，在线才能收） | ✅ 服务器存储 | ✅ 服务器存储 |
| **数据归属** | ✅ 完全由用户自己持有 | ❌ 平台持有 | ❌ 服务器持有 |
| **运行环境** | ✅ 打包为单 exe，无需 Node.js | 需安装 App | 需自建服务 |

> **核心优势**：  
> - **真正的去中心化**：无任何中心节点，所有节点平等。  
> - **极简部署**：一个可执行文件，双击即用，无需配置服务器或数据库。  
> - **隐私优先**：所有消息端到端加密，服务提供者无法窥探。  
> - **灵活连接**：既支持公网 IP 直连，也支持 Tailscale 安全隧道，适合内网穿透。

## 快速开始（给测试朋友）

### 1. 下载并运行
- 从发布页面下载 `p2p-chat.exe`
- 双击运行，首次使用会提示**设置密码**（用于加密本地私钥）
- 程序自动在 `data/chat.db` 生成本地数据库，保存身份和消息

### 2. 基本命令
| 命令 | 说明 |
|------|------|
| `/create <群组名>` | 创建新群组，并自动切换进去 |
| `/join <群组ID>` | 申请加入已有群组（需要管理员批准） |
| `/use <群组ID>` | 切换到指定群组 |
| `/topic <话题名>` | 在当前群组创建/切换话题 |
| `/approve <申请人公钥> <群组ID>` | 管理员批准加入请求 |
| `/list` | 列出所有群组 |
| `/members <群组ID>` | 查看群组成员 |
| `/msgs [群组ID] [话题ID]` | 显示最近20条消息 |
| `/connect <IP> <端口>` | 手动连接对等节点（支持 Tailscale IP） |
| `/tailscale` | 显示本机 Tailscale IP（需安装并登录 Tailscale） |
| `/exit` | 退出 |

### 3. 连接其他用户
- **公网直连**：双方知道对方公网 IP 和 UDP 端口（默认自动分配），使用 `/connect` 连接。
- **Tailscale 方式**（推荐）：双方安装 Tailscale，获得 100.x.x.x 内网 IP，使用 `/connect 100.x.x.x <端口>` 连接，安全且无需端口映射。

>  首次启动后，程序会自动通过 DHT 公告你创建的群组，其他用户可通过 DHT 查找并加入。

## 注意事项
端口要求：UDP 端口需要能被对端访问（NAT 需做端口映射，或使用 Tailscale）。

消息可靠性：UDP 可能丢包，当前版本未实现重传，适合局域网或稳定网络。

私钥安全：密码仅用于本地加密私钥，不会上传，请务必牢记。

不支持离线消息：纯 P2P，对方不在线则消息无法送达。
